## PART 1: THE AGENTIC PROTOCOL (CONSTITUTION)

### 1. THE CASCADE RESEARCH ARCHITECTURE
We utilize a two-layer agentic approach to ensure research depth and validation.

**Layer 1: The "Spotter" Agent (Reconnaissance)**
* Goal: Broad scan of the codebase/market to flag technical debt and opportunities.
* Outputs: Findings Log (Markdown/CSV), Top 3 High-Impact anomalies.

**Layer 2: The "Deep-Diver" Agent (Psychological/Technical Specialist)**
* Goal: Validate "Spotter" findings, perform deep-root analysis, and synthesize ROI.
* Psychological Validation: Apply Story-Verify-Research-Solution framework.

---

### 2. THE STORY-VERIFY-RESEARCH-SOLUTION FLOW
Simpee, every module must pass through this narrative and technical filter:

1. STORY (The Pain): What is the client narrative?
2. VERIFY (The Cultural/Age Factor): Cultural/Age psych context.
3. RESEARCH PLAN: How we verified the pain.
4. SOLUTION: Technical execution plan.
5. ROI ESTIMATION: Measured by concrete criteria.

import json

class ResearchAgent:
    def __init__(self, role):
        self.role = role # 'Spotter' or 'Deep-Diver'

    def execute_research(self, module):
        if self.role == 'Spotter':
            return self._scan_anomalies(module)
        elif self.role == 'Deep-Diver':
            return self._validate_and_analyze(module)

    def _scan_anomalies(self, module):
        # Scan for high-impact technical debt
        return {"action": "Flagging", "scope": "Broad", "status": "Ready"}

    def _validate_and_analyze(self, module):
        # Psychological Validation + Root Cause Analysis
        return {"action": "Root Cause Analysis", "psychology_layer": "Enabled", "status": "Validated"}
